Testing and Debugging

This week we learned about testing and debuggin in javascript. There are different tpyes of errors which will determine how we go about the process. Here are the types of errors you can encounter:

  1. Syntax Errors - errors that are detected by the compiler
  2. Runtime Errors - errors that cause a program to terminate abnormally
  3. Logic Errors - occer when a program does not perform the way it was intended to

It's important to be able to recognize what errors we have so that we can properly test and debug them.

Concole.log()

One debugging method is console.log(). Using this allows us to display a value in the console so we can see what value is being claculated and can see if it's our desired outcome.

a = 5;
b = 6;
c = a + b;
console.log(c);

By doing this we will see that c is equal to 11 and and know whether or not our program is working correctly.

Breakpoints

Another method is by setting breakpoints. We can set breakpoints and when our code runs into one, it will stop executing and we can see the current values. We can then resume the program.

Debugger Keyword

The debugger keyword is similar to settting a breakpoint. It stops the execution of the program where it is placed before continuing.

var x = 15 * 5;
debugger;
document.getElementById("demo").innerHTML = x;

AJAX

W3Schools says that AJAX is a developer's dream for these reasons:

  1. It can update a web page without reloading the page
  2. Request data from a server - after the page has loaded
  3. Receive data from a server - after the page has loaded
  4. Send data to a server - in the background

AJAX stands for Aynchronous JavaScript And XML
There are 7 steps to on how AJAX works:

  1. An event occurs in a web page (the page is loaded, a button is clicked)
  2. An XMLHttpRequest object is created by JavaScript
  3. The XMLHttpRequest object sends a request to a web server
  4. The server processes the request
  5. The server sends a response back to the web page
  6. The response is read by JavaScript
  7. Proper action (like page update) is performed by JavaScript

CSS transitions allow you to animate something that may typically happen immediately. You can customize how it is animated and for how long.

For example, if you want a button to change color when you hover over it, you can. Below is the code to do that.

div {
width: 100px;
height: 100px;
color: red;
transition: color 2s;
}
div:hover {
color: blue;
}

We first target the div in order to style it and give it a red color. When we target it again but use the :hover, it makes it transition to blue when the user hovers over it. The transition also has a clarification of 2s so that the transition happens over a span of 2 seconds.
